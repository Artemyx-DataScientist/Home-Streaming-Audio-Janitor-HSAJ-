# ARCHITECTURE

## Общая идея
Проект разделён на два самостоятельных процесса:
- **Bridge (Node.js)** — отвечает за подключение к внешним источникам событий, нормализацию и доставку в ядро.
- **Core (Python)** — принимает нормализованные события и применяет доменные правила (first_seen, иммунитет Dolby Atmos и др.).

## Поток данных (dev)
1. Bridge поднимается как сервис, готовый подписываться на внешние события и выводит лог старта.
2. Core запускается отдельно и ожидает событий (пока — заглушки с логами).
3. В реальном окружении bridge будет пушить события в core по IPC/HTTP/queue, но скелет оставляет интерфейс открытым для интеграции.

## Девелопмент
- Изоляция зависимостей: `npm` для bridge, виртуальное окружение Python для core.
- Линтеры и тесты выполняются раздельно, но в единой CI-цепочке.

## Будущее расширение
- Добавление настоящей коммуникации между bridge и core.
- Расширение моделей данных с учётом first_seen и статусов качества.
- Поддержка дополнительных ADR по мере уточнения требований.
