# ADR 0003 — Хранение `first_seen`

- Статус: Принят
- Контекст: Нужно отслеживать момент первого появления сущностей (треки, устройства, пользователи) для аналитики и дедупликации. События могут приходить из разных источников с лагом.
- Решение: Включить поле `first_seen` в канонический формат событий и сохранять его на уровне core. При повторных событиях значение не перезаписывается.
- Последствия: Источники должны передавать `first_seen`, либо bridge вычисляет его при первом получении; в хранилище core поле неизменно после первого заполнения; тесты должны проверять сохранение первого значения.
